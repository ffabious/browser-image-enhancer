# Канонический контракт цветовой математики

Этот документ — **единственный источник истины** для математики коррекции изображения.
Он реализован дважды и обязан совпадать побайтово (допуск ±1 младший бит на канал):

- `ml/src/imageenh/colormath.py` — применяется при обучении модели (Python/NumPy);
- `web/src/lib/apply/enhance.ts` — применяется в браузере (TypeScript).

Совпадение проверяется golden-тестами: `ml/src/imageenh/export_golden.py` генерирует
фикстуры (случайные RGBA-массивы + тройки коэффициентов + ожидаемый результат),
`web/tests/colormath.golden.test.ts` сверяет реализацию TS с ними.
**Любое изменение этого документа требует синхронного изменения обеих реализаций
и перегенерации фикстур.**

## Область определения

- Все вычисления ведутся в нормализованном sRGB: `x = v / 255 ∈ [0, 1]`,
  где `v` — 8-битное значение канала. Гамма-линеаризация **не** выполняется
  (работаем в дисплейном пространстве, как PIL/torchvision).
- Альфа-канал не изменяется.
- Промежуточные значения **не** ограничиваются диапазоном — единственный clamp
  выполняется в самом конце.

## Параметры коррекции

Три мультипликативных коэффициента, `1.0` = тождественное преобразование:

| Параметр | Символ | Диапазон |
|---|---|---|
| Яркость (brightness) | β | [0.5, 2.0] |
| Контрастность (contrast) | γ | [0.5, 2.0] |
| Цветность/насыщенность (saturation) | σ | [0.4, 2.5] |

Модель выдаёт вектор `o = (o₀, o₁, o₂) ∈ [−1, 1]³` (выход tanh).
Отображение в коэффициенты (логарифмическая параметризация):

```
β = exp(o₀ · ln 2)      # 2^o₀  ∈ [0.5, 2.0]
γ = exp(o₁ · ln 2)      # 2^o₁  ∈ [0.5, 2.0]
σ = exp(o₂ · ln 2.5)    # 2.5^o₂ ∈ [0.4, 2.5]
```

## Порядок применения (фиксированный)

**яркость → контрастность → цветность**, затем один финальный clamp и квантование.

Для каждого пикселя `(R, G, B)`, `x ∈ {R, G, B}`:

1. **Яркость** (усиление): `x₁ = x · β`
2. **Контрастность** (фиксированная опорная точка 0.5 — намеренно *не* среднее
   по изображению, чтобы Python и TS не зависели от общей статистики):
   `x₂ = (x₁ − 0.5) · γ + 0.5`
3. **Цветность** (люма Rec.601 от *текущего* пикселя, вычисленная из
   неограниченных значений шага 2):
   `Y = 0.299·R₂ + 0.587·G₂ + 0.114·B₂`
   `x₃ = Y + (x₂ − Y) · σ`
4. **Финальный clamp и квантование** (округление half-up, реализуется как
   floor, что одинаково в JS `Math.round` для неотрицательных чисел и в NumPy):
   `v_out = floor(min(max(x₃, 0), 1) · 255 + 0.5)`

Шаги 1–2 — единое аффинное преобразование на канал:
`x₂ = x·β·γ + (0.5 − 0.5·γ)`. Это позволяет в рантайме TS свернуть их в
таблицу из 256 значений `Float32Array` (LUT), а смешивание с люмой (шаг 3)
выполнять попиксельно в float с единственным финальным clamp.

## Требования к реализациям

- **Python**: вычисления в `float32` (NumPy), вход/выход `uint8`.
  Квантование: `np.floor(np.clip(x, 0, 1) * 255 + 0.5).astype(np.uint8)`.
  Запрещено использовать `np.round` (банковское округление).
- **TypeScript**: LUT шагов 1–2 хранится как `Float32Array[256]`
  (`lut[v] = (v/255)·β·γ + 0.5·(1 − γ)`), шаг 3 — в обычных числах JS,
  квантование `Math.min(Math.max(x, 0), 1) * 255 + 0.5 | 0` либо `Math.floor`.
  Обработка ведётся по тайлам ~1 Мпк **на месте** (in-place) в RGBA-буфере.
- Допуск golden-тестов: |Δv| ≤ 1 на канал (расхождения float32/float64).

## Деградация при обучении

Синтетическая деградация в `dataset.py` использует **ту же** функцию `apply`
с сэмплированными коэффициентами деградации (см. `docs/spec.md`, §обучение).
Модель учится предсказывать `o`, при котором `apply(degraded, factors(o))`
максимально близко к оригиналу (L1 в пространстве изображения).
