# Журнал архитектурных решений (ADR)

## ADR-1: Собственный инференс-движок вместо TensorFlow.js

**Решение:** инференс CNN реализован вручную на TypeScript (~400 строк,
5 типов операций), обучение — в Keras/TensorFlow.

**Причины:** модель крошечная (~198 тыс. параметров, один инференс на задачу,
~50 М умножений ≈ 0.1–0.5 с на CPU) — GPU-ускорение не нужно; TF.js добавил бы
~3 МБ к бандлу при явно оцениваемом критерии «оптимизация размера решения»;
CPU-математика идентична во всех браузерах (нет WebGL-квирков Safari).

**Запасной вариант:** модель обучается в Keras, поэтому
`tensorflowjs_converter` + TF.js подключаются без переобучения.

## ADR-2: Применение коррекции — чистый JS (LUT), без WebGL

Слитый проход LUT(яркость+контраст) + попиксельная цветность по 15 Мпк
занимает ~0.3–1 с в worker — в бюджете. WebGL в OffscreenCanvas внутри
worker — лишняя поверхность совместимости. OffscreenCanvas используется
только с контекстом 2d (декодирование и кодирование).

## ADR-3: Фиксированная опорная точка контраста 0.5

Контраст применяется относительно константы 0.5, а не среднего по
изображению: исключает общую статистику между Python (обучение) и TS
(рантайм) и делает шаги яркость+контраст единым аффинным преобразованием
(одна LUT на 256 значений).

## ADR-4: Собственный запасной BMP-декодер

`createImageBitmap` для BMP-блобов не гарантирован во всех движках.
Чистый TS-декодер несжатых BMP (~150 строк) устраняет неопределённость
при почти нулевой цене размера.

## ADR-5: HEIC через ленивый libheif, с нативной пробой

Safari декодирует HEIC нативно (системный декодер). Остальные — нет.
Стратегия: сниффинг `ftyp`-бренда → одноразовая проба нативного
декодирования → при неудаче ленивая загрузка WASM-чанка libheif
(скачивается только при первой HEIC-задаче, в основной бандл не входит).

## ADR-6: Отмена без SharedArrayBuffer

GitHub Pages не позволяет выставить COOP/COEP, поэтому SAB недоступен.
Отмена реализована через macrotask-yield между тайлами и стадиями
конвейера; задержка отмены ≤ ~100 мс вне нативных вызовов. Принято и
задокументировано в ТЗ.

## ADR-7: Предобработка инференса — сжатие 224×224 без сохранения пропорций

Сжатие сохраняет глобальную статистику, нужную модели, и исключает
политику кадрирования, которая могла бы разойтись между Python и TS.
Билинейная интерполяция браузера не побитово равна PIL — модель обучена
с шумом/JPEG-аугментацией для устойчивости; послойные golden-тесты NN
используют фиксированные тензоры, а не результат ресайза.

## Проверка совместимости (M0)

Проверено автоматически пробной страницей (`probe.html`) через Playwright
на движках всех четырёх целевых браузеров (WebKit ≈ Safari,
Chromium ≈ Chrome/Яндекс, Gecko/Firefox ≈ Zen), июнь 2026:

| Утверждение | WebKit (Safari) | Chromium (Chrome/Яндекс) | Firefox (Zen) |
|---|---|---|---|
| Module workers | ✓ | ✓ | ✓ |
| OffscreenCanvas 2d + convertToBlob в worker | ✓ | ✓ | ✓ |
| createImageBitmap(PNG Blob) | ✓ | ✓ | ✓ |
| createImageBitmap(BMP Blob) | ✓ | ✓ | ✓ |
| Нативное декодирование HEIC | ✓ | ✗ → libheif | ✗ → libheif |
| Canvas ≥ 15 Мпк (5000×3000) | ✓ | ✓ | ✓ |

E2E-конвейер (все форматы, отмена, 15 Мпк < 30 с) — 27/27 тестов на трёх
движках (`web/e2e/`). Для финальной сдачи рекомендуется ручной прогон
`probe.html` в реальных установленных Safari, Zen и Яндекс Браузере.
