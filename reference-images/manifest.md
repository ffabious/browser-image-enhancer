# Эталонный пул изображений

Пул формируется скриптом `scripts/make-reference-pool.py` из валидационного
набора **DIV2K** (NTIRE 2017; набор распространяется для исследовательских и
учебных целей) после `scripts/fetch-dataset.sh`.

Каждая запись пула — пара файлов:

- `NNNN-original.png` — исходное («идеальное») изображение, уменьшенное до
  ≤1280 px по длинной стороне;
- `NNNN-<категория>.jpg` — деградированная версия, полученная канонической
  цветовой математикой (`docs/color-math.md`) с задокументированными
  коэффициентами (см. `manifest.json`).

Категории покрывают основные дефекты: недоэкспонирование, переэкспонирование,
низкий контраст, дымка, обесцвеченность, перенасыщенность, ночная сцена,
смешанные дефекты, а также три «идеальных» изображения для проверки
identity safety (система не должна портить хорошие снимки).

Поскольку деградация выполнена известными коэффициентами, оценка качества
(`ml/src/imageenh/evaluate.py`) измеряет не только PSNR/SSIM восстановления,
но и точность восстановления самих параметров (MAE в лог-пространстве).

Стресс-кейсы по размеру/форматам (15 Мпк JPEG/PNG, BMP, HEIC) генерируются
отдельно: `scripts/make-fixtures.py` → `web/e2e/fixtures/`.
